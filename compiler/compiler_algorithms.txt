COMPILER ALGORITHMS

1. LEXICAL ANALYZER ALGORITHM

Input:
Source program as a character stream.

Output:
Sequence of tokens.

Algorithm:
1. Set i = 0.
2. While i < length(input):
3. If input[i] is whitespace, increment i and continue.
4. If input[i] begins a comment, consume the entire comment, update i, and continue.
5. Set state = start_state, j = i, last_accept = none.
6. While j < length(input) and DFA transition exists on input[j]:
7. Move to the next DFA state.
8. Increment j.
9. If the new state is accepting, set last_accept = (token_type, j).
10. If last_accept exists:
11. Extract lexeme = input[i ... last_accept.position - 1].
12. If token_type is identifier and lexeme is in keyword table, change token_type to keyword.
13. Emit (token_type, lexeme).
14. Set i = last_accept.position.
15. Otherwise report lexical error at position i and stop or skip the invalid character.
16. After the scan ends, emit EOF.

Pseudocode:
while i < n
    if input[i] is whitespace
        i = i + 1
        continue
    if input[i] starts comment
        skip comment
        continue
    state = start
    j = i
    last_accept = none
    while transition exists from state on input[j]
        state = transition(state, input[j])
        j = j + 1
        if state is accepting
            last_accept = (state, j)
    if last_accept exists
        emit token from input[i ... last_accept - 1]
        i = last_accept
    else
        report lexical error
emit EOF

Example:
Input string:
id + id * id

Token sequence:
id, +, id, *, id


2. LEFT FACTORING ALGORITHM

Input:
Grammar G.

Output:
Equivalent grammar with common prefixes factored.

Algorithm:
1. For each nonterminal A, collect all productions of A.
2. Find a largest common prefix alpha shared by two or more alternatives of A.
3. If no such prefix exists, leave A unchanged.
4. Otherwise create a new nonterminal A'.
5. Replace all productions A -> alpha beta1 | alpha beta2 | ... | alpha betak by:
6. A -> alpha A'
7. A' -> beta1 | beta2 | ... | betak
8. If any betai is empty, replace it by epsilon.
9. Keep the remaining productions of A unchanged.
10. Repeat for every nonterminal until no nonterminal has two alternatives with a common prefix.

Pseudocode:
repeat
    changed = false
    for each nonterminal A
        find a common prefix alpha among productions of A
        if alpha exists
            create new nonterminal A'
            replace A -> alpha beta1 | alpha beta2 | ... by
                    A -> alpha A'
                    A' -> beta1 | beta2 | ...
            changed = true
until changed = false

Example:
Given grammar:
E -> E + T | T
T -> T * F | F
F -> (E) | id

Observation:
No nonterminal has alternatives with the same left prefix.
So left factoring is not needed.

Result:
E -> E + T | T
T -> T * F | F
F -> (E) | id


3. LEFT RECURSION REMOVAL ALGORITHM

Input:
Grammar G with nonterminals A1, A2, ..., An.

Output:
Equivalent grammar without left recursion.

Algorithm:
1. Order the nonterminals as A1, A2, ..., An.
2. For i = 1 to n:
3. For j = 1 to i - 1:
4. Replace every production of the form Ai -> Aj gamma by:
5. Ai -> delta1 gamma | delta2 gamma | ... | deltam gamma
6. where Aj -> delta1 | delta2 | ... | deltam are the current productions of Aj.
7. After all such replacements, remove direct left recursion from Ai.
8. To remove direct left recursion, split productions of Ai into:
9. Left-recursive: Ai -> Ai alpha1 | Ai alpha2 | ... | Ai alpham
10. Non-left-recursive: Ai -> beta1 | beta2 | ... | betan
11. If m = 0, do nothing.
12. Otherwise create a new nonterminal Ai'.
13. Replace Ai by:
14. Ai -> beta1 Ai' | beta2 Ai' | ... | betan Ai'
15. Ai' -> alpha1 Ai' | alpha2 Ai' | ... | alpham Ai' | epsilon
16. Repeat until all nonterminals are processed.

Pseudocode:
for i = 1 to n
    for j = 1 to i - 1
        replace each Ai -> Aj gamma
        using productions of Aj
    separate Ai productions into:
        Ai -> Ai alpha   and   Ai -> beta
    if direct left recursion exists
        create Ai'
        replace with:
            Ai  -> beta Ai'
            Ai' -> alpha Ai' | epsilon

Example:
Given grammar:
E -> E + T | T
T -> T * F | F
F -> (E) | id

After removing left recursion:
E  -> T E'
E' -> + T E' | epsilon
T  -> F T'
T' -> * F T' | epsilon
F  -> (E) | id


4. AMBIGUITY REMOVAL ALGORITHM

Note:
There is no general algorithm that removes ambiguity for every context-free grammar.
The following is the practical procedure used for common programming-language grammars.

Algorithm:
1. Find a string that has more than one parse tree or more than one leftmost/rightmost derivation.
2. Identify the source of ambiguity:
3. Operator precedence ambiguity.
4. Operator associativity ambiguity.
5. Optional-part ambiguity such as dangling else.
6. If the ambiguity is from operators, divide operators into precedence levels.
7. Create one nonterminal for each precedence level.
8. For left-associative operators at a level L, use:
9. L -> L op N | N
10. For right-associative operators at a level L, use:
11. L -> N op L | N
12. For non-associative operators at a level L, allow only one occurrence at that level.
13. Put operands, identifiers, numbers, and parenthesized expressions in the lowest-level nonterminal.
14. If the ambiguity is from unmatched constructs, split the grammar into matched and unmatched forms.
15. Replace the ambiguous productions by the new structured productions.
16. Recompute parse trees for the previously ambiguous strings.
17. Repeat until each valid string has exactly one parse tree.

Pseudocode:
find ambiguous string w
while w has more than one parse tree
    identify cause of ambiguity
    if cause is operator precedence
        create one nonterminal per precedence level
    else if cause is associativity
        rewrite grammar for left or right associativity
    else if cause is dangling else or similar
        split grammar into matched and unmatched forms
    test grammar again on w
    if grammar is no longer ambiguous
        stop

Example:
Ambiguous grammar:
E -> E + E | E * E | (E) | id

Ambiguity-free grammar using precedence and associativity:
E -> E + T | T
T -> T * F | F
F -> (E) | id

Meaning:
* has higher precedence than +
Both + and * are left associative
So id + id * id is parsed as id + (id * id)


5. FIRST SET COMPUTATION ALGORITHM

Input:
Grammar G.

Output:
FIRST set for every grammar symbol.

Algorithm:
1. For every terminal a, set FIRST(a) = {a}.
2. Set FIRST(epsilon) = {epsilon}.
3. For every nonterminal A, initialize FIRST(A) = empty set.
4. Repeat until no FIRST set changes:
5. For each production A -> X1 X2 ... Xk:
6. Set all_nullable = true.
7. For p = 1 to k:
8. Add FIRST(Xp) - {epsilon} to FIRST(A).
9. If epsilon is not in FIRST(Xp), set all_nullable = false and stop scanning this production.
10. If all_nullable is true after scanning all symbols, add epsilon to FIRST(A).

Equivalent sequence form:
1. To compute FIRST(X1 X2 ... Xk), start with empty set.
2. Add FIRST(X1) - {epsilon}.
3. If epsilon is in FIRST(X1), also add FIRST(X2) - {epsilon}.
4. Continue this way from left to right.
5. If every Xi can derive epsilon, add epsilon.

Pseudocode:
for each terminal a
    FIRST(a) = {a}
for each nonterminal A
    FIRST(A) = empty
repeat
    changed = false
    for each production A -> X1 X2 ... Xk
        for p = 1 to k
            add FIRST(Xp) - {epsilon} to FIRST(A)
            if epsilon not in FIRST(Xp)
                break
        if epsilon is in FIRST(X1)...FIRST(Xk)
            add epsilon to FIRST(A)
until changed = false

Example:
Using the left-recursion-free grammar:
E  -> T E'
E' -> + T E' | epsilon
T  -> F T'
T' -> * F T' | epsilon
F  -> (E) | id

FIRST(F)  = { (, id }
FIRST(T)  = { (, id }
FIRST(T') = { *, epsilon }
FIRST(E)  = { (, id }
FIRST(E') = { +, epsilon }


6. FOLLOW SET COMPUTATION ALGORITHM

Input:
Grammar G with start symbol S.

Output:
FOLLOW set for every nonterminal.

Algorithm:
1. For every nonterminal A, initialize FOLLOW(A) = empty set.
2. Add $ to FOLLOW(S).
3. Repeat until no FOLLOW set changes:
4. For each production A -> alpha B beta:
5. Add FIRST(beta) - {epsilon} to FOLLOW(B).
6. If beta is empty, add FOLLOW(A) to FOLLOW(B).
7. If epsilon is in FIRST(beta), add FOLLOW(A) to FOLLOW(B).
8. Continue until all productions are processed and no set changes.

Equivalent scan method for a production A -> X1 X2 ... Xn:
1. Scan symbols from left to right.
2. Whenever Xi is a nonterminal, let beta = Xi+1 Xi+2 ... Xn.
3. Add FIRST(beta) - {epsilon} to FOLLOW(Xi).
4. If beta is empty or epsilon is in FIRST(beta), add FOLLOW(A) to FOLLOW(Xi).

Pseudocode:
for each nonterminal A
    FOLLOW(A) = empty
FOLLOW(StartSymbol) = {$}
repeat
    changed = false
    for each production A -> X1 X2 ... Xn
        for each Xi that is a nonterminal
            beta = Xi+1 ... Xn
            add FIRST(beta) - {epsilon} to FOLLOW(Xi)
            if beta is empty or epsilon in FIRST(beta)
                add FOLLOW(A) to FOLLOW(Xi)
until changed = false

Example:
Using the same grammar:
E  -> T E'
E' -> + T E' | epsilon
T  -> F T'
T' -> * F T' | epsilon
F  -> (E) | id

FOLLOW(E)  = { ), $ }
FOLLOW(E') = { ), $ }
FOLLOW(T)  = { +, ), $ }
FOLLOW(T') = { +, ), $ }
FOLLOW(F)  = { *, +, ), $ }
